    Headroom Security — Enterprise DLP & Guardrails Plan¶

    Overview¶

    Enterprise-only security layer that plugs into Headroom as a separate private package (headroom-security). Provides PII detection/redaction, action guardrails, and audit logging — gated behind a license key.

    Business model: Open-core. OSS Headroom = compression. Enterprise Headroom = compression + security.

    Architecture¶

    OSS headroom (public repo)                    headroom-security (private repo)
┌─────────────────────────────┐              ┌──────────────────────────────┐
│  Proxy Pipeline:            │              │  SecurityPipeline:           │
│                             │   imports    │                              │
│  request → [SecurityHook]──────────────────→  PII Scanner (Presidio +    │
│          → [Compression]    │              │    GLiNER + PasteProof)      │
│          → [Forward to LLM] │              │  Action Guardrails           │
│                             │              │  Audit Logger                │
│  response → [SecurityHook]──────────────────→  Response PII check        │
│           → [Return]        │              │  De-anonymize (CCR-based)    │
│                             │              │                              │
│  License check: if key →    │              │  License validator           │
│    activate security hooks  │              │  Policy engine (YAML)        │
└─────────────────────────────┘              └──────────────────────────────┘

    Plugin Interface (OSS side — ~10 lines)¶

    # In OSS headroom/proxy/server.py
self.security = None
try:
    from headroom_security import SecurityPipeline
    self.security = SecurityPipeline(config)
    if not self.security.is_licensed():
        self.security = None
except ImportError:
    pass  # OSS mode

    Pipeline Ordering¶

    Security runs BEFORE compression (scan real content, not compressed):

    Request:  Parse → SECURITY SCAN → Compress → Forward to LLM
Response: Receive → SECURITY SCAN → De-anonymize → Return to client

    Scanner Stack¶

    Model Purpose F1 Size Latency (CPU)
    Presidio (framework) Orchestrator — score fusion, span grounding, reversible anonymization N/A N/A N/A
    GLiNER-PII-large (ONNX) 60+ entity types, zero-shot (custom entities at runtime) 0.83 197MB ~75ms
    PasteProof v2 (ModernBERT) Code/config/secrets — Samsung-type leak prevention 0.97 ~400MB ~30ms
    Local LLM (Ollama) Ambiguous/context-dependent PII (async, optional) Context-dependent Varies Seconds
    Custom recognizers Enterprise-specific patterns (project names, internal IDs) User-defined N/A <1ms

    Why Not Regex/NER Alone¶

    • Regex misses context-dependent PII (name that's also a common word)
    • Basic NER has poor recall on non-Western names and domain-specific entities
    • GLiNER's zero-shot capability = admins add entity types at runtime without retraining
    • PasteProof catches secrets in code that regex patterns don't cover

    Reversible Anonymization (CCR-Based)¶

    Unique Headroom advantage — uses existing CCR infrastructure:

    Input:  "John Smith's SSN is 123-45-6789, call him at john@acme.com"
                    ↓ Anonymize
Sent:   "[PERSON_a7b3]'s SSN is [SSN_c2d4], call them at [EMAIL_e5f6]"
                    ↓ CCR store
CCR:    {a7b3: "John Smith", c2d4: "123-45-6789", e5f6: "john@acme.com"}
                    ↓ LLM responds
LLM:    "I've noted [PERSON_a7b3]'s information..."
                    ↓ De-anonymize
Output: "I've noted John Smith's information..."

    LLM never sees real PII. Response re-expansion is lossless. GDPR gold standard.

    Action Guardrails¶

    • Tool call whitelist/blacklist (block file_delete, git_push_force, DROP TABLE)
    • Topic blocking (litigation, M&A, competitor names)
    • Rate limiting per user/team
    • Tiered approval (low/medium/high risk actions)

    Policy Config (YAML)¶

    security:
  pii:
    enabled: true
    action: redact  # redact | block | alert
    entities: [credit_card, ssn, email, api_key, person_name]
    custom_patterns:
      - name: employee_id
        regex: "EMP-[0-9]{6}"
      - name: internal_url
        regex: "https?://[^/]*\\.internal\\.corp\\.com"
  guardrails:
    blocked_tools: [file_delete, git_push_force, drop_table]
    blocked_topics: [litigation, merger_acquisition]
    max_tokens_per_minute: 100000
  audit:
    enabled: true
    log_redacted: true
    destination: file  # file | webhook | syslog

    License Gating¶

    HEADROOM_LICENSE_KEY env var → JWT with embedded claims
  - Offline validation (embedded expiry + feature flags)
  - Periodic online refresh via headroomlabs.ai/api/license/verify
